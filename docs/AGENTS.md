# Docs Agent Instructions

## Scope

This directory owns product, architecture, operations, deployment, import compatibility, and learning documentation.

## Docs Rules

- Write docs for a developer coming from Python/Django.
- Keep language concrete and implementation-oriented.
- Document the current behavior, not only future intentions.
- When a decision changes architecture, add or update an ADR.

## Required Topics

- Product scope and non-goals.
- Go architecture.
- SQLite schema and migrations.
- Solidtime import compatibility.
- Docker-first operations.
- Observability and performance measurement.
- Email notifications and in-process scheduler (`docs/29-email-notifications.md`).
